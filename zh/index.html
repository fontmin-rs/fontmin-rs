fontmin-rs高性能字体压缩与转换工具链

用 Rust 处理字体核心逻辑，通过 Node.js binding 暴露现代前端工作流需要的 CLI 和 TypeScript API。

快速开始
Node API

字体子集化

按 text、textFile、unicodes 或 basicText 保留实际需要的字形，并支持 conservative/preserve 等布局保留策略。

多格式输出

当前支持 TTF、WOFF、WOFF2、EOT、SVG font 和 @font-face CSS 输出，适合现代 Web 与遗留兼容场景。

Rust + Node

核心 crate、CLI、N-API binding 与 TypeScript 包共享一套行为目标，既能跑命令，也能嵌入构建脚本。

当前状态 ​

fontmin-rs 仍处在早期实现阶段，但已经具备一条可用的字体处理链路：

  • fontmin-rs subset 按文本裁剪 TTF。
  • fontmin-rs convert 在 TTF、WOFF、WOFF2、EOT、SVG font 等格式间转换。
  • fontmin-rs build 从输入字体批量生成多格式产物和 CSS。
  • fontmin-rs inspect 输出 TTF、OTF、WOFF、EOT 等字体元信息。
  • fontmin-rs init 创建初始 JSONC 配置文件。
  • fontmin-rs npm 包提供同名 bin、低层 native helpers、optimize(config) pipeline 和 Fontmin-compatible chain。
